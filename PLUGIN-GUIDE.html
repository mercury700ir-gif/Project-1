۱. معرفی سیستم پلاگین

سیستم پلاگین این سایت مشابه WordPress عمل می‌کند. پلاگین‌ها می‌توانند به هسته سایت متصل شوند و امکانات جدیدی اضافه کنند.

نکته مهم: هر پلاگین قبل از فعال شدن توسط موتور سایت بررسی می‌شود. در صورت وجود مشکل امنیتی یا عملکردی، اجازه فعال شدن داده نمی‌شود.

ویژگی‌های کلیدی سیستم پلاگین:

  • جدول‌های اختصاصی برای هر پلاگین (قابل پاکسازی)
  • بررسی خودکار کد قبل از نصب
  • سازگاری با ساختار database سایت
  • عدم کاهش سرعت سایت
  • عدم ایجاد خطا در فرآیندهای سایت

۲. ساختار فایل پلاگین

هر پلاگین باید در یک پوشه جداگانه با ساختار زیر قرار گیرد:

my-plugin/
├── plugin.json          ← فایل manifest (الزامی)
├── index.js             ← فایل اصلی پلاگین (الزامی)
├── style.css            ← استایل‌های اختصاصی (اختیاری)
├── settings.json        ← تنظیمات پیش‌فرض (اختیاری)
└── assets/              ← فایل‌های رسانه (اختیاری)
    ├── logo.png
    └── ...

۳. فایل Manifest (JSON)

هر پلاگین باید یک فایل plugin.json داشته باشد:

{
  "name": "my-plugin",
  "title": "عنوان پلاگین",
  "version": "1.0.0",
  "description": "توضیحات کوتاه درباره عملکرد پلاگین",
  "author": "نام توسعه‌دهنده",
  "main": "index.js",
  "settings": [
    { "key": "api_key", "label": "کلید API", "type": "text", "default": "" },
    { "key": "enable", "label": "فعال‌سازی", "type": "boolean", "default": true }
  ],
  "permissions": ["manage_options", "edit_posts"],
  "dependencies": [],
  "database_tables": ["my_plugin_data"]
}
فیلدهای manifest:
  • name: شناسه یکتای پلاگین (انگلیسی، بدون فاصله)
  • title: عنوان نمایشی فارسی
  • version: شماره نسخه (فرمت semver)
  • main: فایل اجرایی اصلی
  • settings: آرایه فیلدهای تنظیمات
  • permissions: دسترسی‌های مورد نیاز
  • database_tables: جداول اختصاصی پلاگین

۴. اتصال به پایگاه داده

هر پلاگین می‌تواند جداول اختصاصی خود را در database سایت ایجاد کند. این جداول باید با پیشوند نام پلاگین شروع شوند:

CREATE TABLE my_plugin_data (
  id INT AUTO_INCREMENT PRIMARY KEY,
  post_id INT,
  data_key VARCHAR(100),
  data_value TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (post_id) REFERENCES posts(id) ON DELETE CASCADE
);
نکته امنیتی: از query‌های آماده (Prepared Statements) استفاده کنید. هرگز مقادیر کاربر را مستقیماً در query قرار ندهید.

۵. سیستم Hook و رویدادها

پلاگین‌ها می‌توانند به رویدادهای سایت متصل شوند:

رویدادتوضیحزمان اجرا
onInitراه‌اندازی اولیههنگام فعال شدن پلاگین
onPostSaveذخیره پستبعد از ذخیره هر پست
onUserRegisterثبت‌نام کاربرهنگام عضویت کاربر جدید
onPageRenderرندر صفحهقبل از نمایش هر صفحه
onCronزمان‌بندیاجرای دوره‌ای

۶. بررسی امنیت و کیفیت

قبل از فعال شدن، پلاگین توسط موتور سایت بررسی می‌شود:

  • بررسی وجود توابع ممنوعه (eval, exec, system)
  • بررسی SQL Injection
  • بررسی XSS
  • بررسی حملات CSRF
  • بررسی سرعت اجرا
  • بررسی سازگاری با database سایت
هشدار: پلاگین‌هایی که کد ناامن داشته باشند، اجازه فعال شدن نخواهند داشت.

۷. نصب و فعال‌سازی

  1. فایل پلاگین (ZIP, RAR, TAR, 7Z) را آپلود کنید
  2. در بخش پلاگین‌های پنل مدیریت، پلاگین را پیدا کنید
  3. روی «فعال» کلیک کنید
  4. تنظیمات پلاگین را پیکربندی کنید
  5. تست کنید که پلاگین بدون خطا کار می‌کند

۸. نمونه کامل پلاگین

نمونه یک پلاگین ساده شمارش بازدید:

// index.js — پلاگین شمارش بازدید
module.exports = {
  name: "view-counter",
  title: "شمارشگر بازدید",

  onInit: function() {
    console.log("پلاگین شمارشگر بازدید فعال شد");
  },

  onPageRender: function(page) {
    var key = "views_" + page.slug;
    var count = this.getSetting(key, 0);
    this.setSetting(key, count + 1);
    page.viewCount = count + 1;
  },

  getSetting: function(key, def) {
    var val = localStorage.getItem("plugin_view-counter_" + key);
    return val ? JSON.parse(val) : def;
  },

  setSetting: function(key, val) {
    localStorage.setItem("plugin_view-counter_" + key, JSON.stringify(val));
  }
};

۹. سوالات متداول

آیا پلاگین‌ها به سرعت سایت آسیب می‌زنند؟

خیر. قبل از فعال شدن، سرعت اجرای پلاگین بررسی می‌شود. پلاگین‌های سنگین اجازه فعال شدن ندارند.

آیا می‌توانم از زبان‌های مختلف استفاده کنم؟

پلاگین‌های اصلی باید با JavaScript نوشته شوند. برای پلاگین‌های پیشرفته‌تر می‌توانید از ماژول‌های سازگار استفاده کنید.

آیا می‌توانم پلاگین را حذف کنم؟

بله. ابتدا پلاگین را غیرفعال کنید، سپس حذف کنید. جداول اختصاصی پلاگین نیز حذف خواهند شد.

چگونه پلاگین را به‌روزرسانی کنم؟

نسخه جدید را آپلود کنید. سیستم به‌طور خودکار نسخه قبلی را جایگزین می‌کند.
