۱. نمای کلی پروژه

سایت شخصی «مهبد نادری» یک پورتفولیوی حرفه‌ای برای یک معمار رشد دیجیتال و مشاور بازاریابی است. این سایت شامل بخش‌های عمومی (سایت اصلی) و یک پنل مدیریت کامل است. تمام صفحات باید lang="fa" و dir="rtl" باشند و فونت فقط IRANSans باشد.

پرامپت اولیه

یک سایت شخصی حرفه‌ای برای «مهبد نادری» با عنوان شغلی «معمار رشد دیجیتال» بساز. سایت شامل صفحات اصلی (خانه، درباره من، تماس، نوشته‌ها، ثبت‌نام) و یک پنل مدیریت کامل با ۹ تب باشد. تمام صفحات باید RTL و فارسی باشند و فونت IRANSans باشد.

۴. الزامات عمومی سایت

پرامپت الزامات عمومی

تمام صفحات سایت باید دارای ویژگی‌های زیر باشند:

Accessibility (دسترسی‌پذیری)

  • لینک Skip to content در بالای هر صفحه
  • ARIA landmarks: role="banner" روی header، role="main" روی main، role="contentinfo" روی footer
  • شناسه id="main-content" روی تگ main
  • استایل‌های :focus-visible با outline آبی ۳ پیکسل
  • پشتیبانی از prefers-reduced-motion
  • ترپ فوکوس در منوی موبایل

Performance (عملکرد)

  • تگ‌های <script> با defer
  • تصاویر زیر fold با loading="lazy"
  • تصویر LCP با fetchpriority="high"
  • <link rel="preconnect"> برای CDN
  • فونت‌ها با font-display: swap

SEO

  • تگ‌های Open Graph (og:type, og:title, og:description, og:image, og:locale)
  • <link rel="canonical">
  • JSON-LD Structured Data (Person schema)
  • <meta name="theme-color">
  • sitemap.xml با تمام صفحات
  • robots.txt با قوانین خزش
  • دقیقاً یک <h1> در هر صفحه

Persian Number System

تابع toPersianNumbers(str) باید اعداد انگلیسی ۰-۹ را به اعداد فارسی ۰-۹ تبدیل کند و در تمام متن‌های قابل مشاهده اعمال شود (به جز inputها، SVGها، لینک‌های tel:/mailto:).

۵. صفحه اصلی (Home)

پرامپت صفحه اصلی

صفحه اصلی سایت شامل دو بخش مجزا برای دسکتاپ و موبایل باشد:

دسکتاپ — Hero Swiper

  • اسلایدر تمام‌صفحه با افکت fade و crossFade
  • ۴ اسلاید با تصاویر موجود (profile-formal.jpeg، consulting-room.jpeg، case-score.jpeg، profile-light.jpeg)
  • هر اسلاید دارای overlay gradient تیره باشد
  • محتوای هر اسلاید: kicker، h2 عنوان، lead پاراگراف، دو دکمه CTA
  • فقط در min-width: 901px نمایش داده شود
  • Auto-play هر ۵ ثانیه با سرعت transition ۹۰۰ms
  • فلش‌های ناوبری و pagination dots سازگار با RTL
  • انیمیشن ورود متن‌ها با stagger

موبایل — Hero با پس‌زمینه

  • تصویر تمام‌عرض با overlay gradient از بالا به پایین
  • بادج «مشاوره تخصصی رایگان» با رنگ طلایی
  • عنوان h1 سفید، lead سفید شفاف
  • دو دکمه تمام‌عرض با backdrop-blur
  • نوار آمار: ۱۰+ سال تجربه | MBA کارلتون | ۱۰۰+ پروژه موفق
  • انیمیشن fadeSlideUp با stagger ۱۲۰ms

بخش‌های زیر Hero

  • Feature Band: رویکرد کاری با گرادینت پس‌زمینه
  • Section Grid: ۳ کارت اطلاعاتی (استراتژی رشد، ساختار مارکتینگ، مشاوره مدیریت)
  • انیمیشن اسکرول با IntersectionObserver
  • پورتره شناور با انیمیشن soft-float

۶. صفحه درباره من

  • Hero compact با h1، lead، دو دکمه، پرتره با کپشن
  • بخش دو ستونه: مسیر حرفه‌ای (feature-band) + لیست فرآیند (۳ مرحله با شماره فارسی)
  • بخش ارزش‌ها: ۳ کارت اطلاعاتی
  • پرتره profile-light.jpeg

۷. صفحه تماس با من

  • Hero compact دو ستونه: پرتره + پنل تماس
  • پنل تماس شامل: عنوان، lead، دکمه تلفن (tel:)، دکمه ایمیل (mailto:)
  • آیتم‌های تماس با آیکون SVG (رنگ rgb(36,80,122))
  • فرم تماس با فیلدها: نام، شماره (dir=ltr)، ایمیل (dir=ltr)، موضوع (select)، پیام
  • ارسال از طریق mailto: link
  • ۳ کارت mini: ارزیابی بازاریابی، طراحی سیستم رشد، همراهی مدیریتی

۸. صفحه نوشته‌ها (وبلاگ)

  • خواندن پست‌ها از localStorage (آرایه admin_blog_posts)
  • پست‌های با وضعیت published نمایش داده شوند
  • لیست پست‌ها با کارت‌ها: عنوان، تاریخ، دسته‌بندی، خلاصه
  • فیلتر sidebar: دسته‌بندی، نویسنده، مرتب‌سازی، نوع محتوا (مقاله/ویدئو)
  • نمایش تکی پست با دکمه بازگشت
  • پشتیبانی از ویدئو در پست‌ها

۹. صفحه ورود و ثبت‌نام

  • تب‌های ورود و ثبت‌نام در یک کارت مرکزی
  • ورود: ایمیل + رمز عبور → بررسی در localStorage
  • ثبت‌نام: نام، شماره موبایل، ایمیل (نام کاربری)، رمز عبور
  • ذخیره کاربران در site_users در localStorage
  • ورود خودکار بعد از ثبت‌نام
  • لینک «فراموشی رمز عبور» → صفحه reset-password.html
  • دکمه «ورود/عضویت» در هدر سایت (دسکتاپ: متن، موبایل: آیکون)

۱۰. پنل مدیریت

پرامپت پنل مدیریت

یک پنل مدیریت کامل مشابه WordPress Admin بساز. شامل sidebar چپ (RTL)، header بالا، ۹ تب مدیریتی، Dark/Light mode، احراز هویت با localStorage، و تمام متن‌ها فارسی و RTL باشد.

ساختار کلی

  • Sidebar: ثابت در دسکتاپ، کشویی در موبایل، با آیکون‌های SVG
  • Top Bar: عنوان صفحه، جستجو، نام کاربر، دکمه بازگشت به سایت
  • احراز هویت: صفحه ورود با نام کاربری admin و رمز admin
  • Dark/Light mode: ذخیره ترجیح در localStorage
  • Responsive: سایدبار در موبایل overlay می‌شود

۹ تب مدیریتی

  1. داشبورد: آمار بازدید، پست‌های اخیر، کاربران فعال
  2. مدیریت محتوا: صفحات + بنرها و تصاویر
  3. مدیریت کاربران: مدیریت سایت + اعضای سایت
  4. گالری: تصاویر + ویدیوها + اطلاعات مetadata
  5. وبلاگ: لیست پست‌ها + ویرایشگر + هوش مصنوعی
  6. فایل‌ها: آپلود + دانلود + حذف
  7. پلاگین‌ها: لیست + آپلود + Plugin Edit Tool
  8. شبکه‌های اجتماعی: اتصال + ربات‌ها + زمان‌بندی
  9. تنظیمات: SEO + Analytics + Tag Manager + تماس

۱۱. مدیریت محتوا و بنرها

ویرایشگر صفحات (مشابه Elementor)

  • بخش‌های قابل جمع‌شدن: Hero، محتوا، SEO، تصویر
  • هر بخش با آیکون و دکمه «ویرایش/بستن»
  • فیلدها: h1، lead، kicker، متن دکمه‌ها، لینک دکمه‌ها
  • ویرایشگر Quill.js برای محتوای اصلی
  • فیلدهای SEO: عنوان، توضیحات متا، کلمات کلیدی، canonical

مدیریت بنرها

  • لیست بنرها با پیش‌نمایش تصویر
  • ویرایش: عنوان، تصویر، لینک، موقعیت نمایش
  • نقطه کانونی: اسلایدر X و Y برای تنظیم object-position
  • انتخاب از گالری: دکمه‌ای که Modal فایل منیجر باز می‌کند
  • فعال/غیرفعال کردن و حذف

۱۲. مدیریت کاربران

سیستم نقش‌ها

نقشدسترسی
مدیر سایت (admin)دسترسی کامل به تمام بخش‌ها
مدیر محتوا (editor)فقط محتوا، گالری، فایل‌ها، وبلاگ، شبکه‌ها
کاربر عادی (subscriber)فقط مشاهده سایت و ثبت‌نام

تب دوگانه کاربران

  • تب مدیریت سایت: لیست مدیران با نقش، افزودن، ویرایش رمز، حذف
  • تب اعضای سایت: لیست کاربران ثبت‌نام‌شده با نام، ایمیل، شماره، تاریخ، تیک خبرنامه
  • ویرایش اطلاعات عضو شامل نام، شماره، رمز جدید، عضویت خبرنامه

۱۳. گالری تصاویر

اطلاعات زیر هر تصویر

  • نام فایل: قابل ویرایش
  • تگ‌ها: قابل ویرایش (با کاما جدا شوند)
  • فرمت: WebP, JPEG, PNG, SVG
  • حجم: با فرمت خوانا (B, KB, MB)
  • ابعاد: عرض × ارتفاع

راهنمای فرمت تصاویر

  • WebP / JPEG: عکس‌ها و تصاویر پیچیده — حجم کمتر، کیفیت بالا
  • PNG: لوگو، گرافیک و شفافیت — بدون افت کیفیت
  • SVG: آیکون‌ها و تصاویر برداری — مقیاس‌پذیر
  • WebP: انیمیشن‌ها — پشتیبانی از GIF متحرک

فایل منیجر

Modal انتخاب تصویر از گالری با نمایش شبکه‌ای تصاویر. کلیک روی تصویر → انتخاب → بازگشت آدرس به فیلد درخواست‌کننده.

۱۴. وبلاگ و هوش مصنوعی

انتشار زمان‌دار

  • دکمه «انتشار» با dropdown: انتشار فوری + انتشار زمان‌دار
  • انتخاب پست + تاریخ/ساعت → وضعیت scheduled
  • بررسی خودکار هر ۳۰ ثانیه برای انتشار پست‌های زمان‌دار شده

ویرایشگر Quill.js

  • تولبار کامل: h1-h3، bold/italic/underline، list، blockquote، code-block، link، image، align
  • فیلد آدرس ویدئو با قابلیت انتخاب از گالری
  • دکمه درج تصویر از گالری

تولید محتوا با هوش مصنوعی (سیمولیشن پیشرفته)

  • ورودی: موضوع یا فایل Word/PDF
  • گزینه‌ها: نوع محتوا (۴ نوع)، لحن (۴ لحن)، تعداد کلمات
  • خروجی: عنوان SEO، توضیحات متا، تگ‌ها، محتوای کامل با ساختار h2/h3، blockquote، لیست
  • رعایت RTL و قواعد نگارشی فارسی
  • دکمه‌ها: کپی، استفاده در ویرایشگر، ذخیره به عنوان پست

۱۵. پلاگین‌ها

آپلود پلاگین

  • فرمت‌های پشتیبانی: ZIP, RAR, TAR, 7Z

Plugin Edit Tool

  • انتخاب پلاگین از dropdown
  • نمایش کد در textarea با استایل monospace
  • دکمه ذخیره / رد تغییرات
  • ذخیره کد در localStorage

تنظیمات پلاگین

  • دکمه تنظیمات → Modal باز شود
  • ذخیره → Modal بسته شود + پیام موفقیت + نمایش سبز زیر پلاگین

راهنمای ساخت پلاگین

۱۶. شبکه‌های اجتماعی

۴ تب

  1. اتصال شبکه‌ها: لیست شبکه‌ها + دستورالعمل اتصال برای هرکدام
  2. ربات‌ها: لیست ربات‌ها + افزودن ربات جدید
  3. زمان‌بندی انتشار: انتخاب پست + اکانت‌های عادی + ربات‌ها + زمان
  4. تاریخچه انتشار: جدول وضعیت انتشارها

دستورالعمل اتصال هر شبکه

  • واتساپ: WhatsApp Business API + Webhook
  • بله: ساخت ربات از @BaleBot
  • روبیکا: پنل توسعه‌دهندگان
  • تلگرام: BotFather + Bot Token
  • اینستاگرام: Facebook Developer + Graph API

۱۷. تنظیمات

  • SEO: عنوان سایت، توضیحات متا، کلمات کلیدی، نقشه سایت
  • Google Analytics: فیلد Tracking ID (فرمت G-XXXXXXXXXX) + دکمه تست اتصال با اعتبارسنجی regex
  • Google Tag Manager: فیلد Container ID (فرمت GTM-XXXXXXX) + دکمه تست اتصال
  • اطلاعات تماس: شماره، ایمیل، آدرس
  • تمام تنظیمات در localStorage ذخیره شوند

۱۸. الزامات فنی و عملکردی

پایگاه داده (database.sql)

اسکیمای MySQL شامل جداول: users, pages, posts, categories, tags, post_tags, banners, media, social_accounts, social_schedules, files, plugins, settings, contact_submissions, activity_log

CSS

  • بریک‌پوینت‌ها: 901px (دسکتاپ/mوبایل)، 900px، 768px، 560px، 480px
  • جدول‌ها در موبایل به layout کارتی تبدیل شوند با data-label
  • Sub-tabs در موبایل wrap شوند (flex-wrap: wrap)
  • Modals در موبایل از پایین slide شوند

JS

  • ذخیره‌سازی: localStorage با پیشوند admin_ برای پنل و site_ برای سایت
  • سیستم Store: Store.get(key, fallback) و Store.set(key, val)
  • IntersectionObserver برای انیمیشن اسکرول و lazy load
  • Quill.js برای ویرایشگر متن غنی
  • Swiper.js برای اسلایدر Hero

۱۹. راه‌اندازی و استقرار

راه‌اندازی محلی

  • فایل index.html را مستقیماً در مرورگر باز کنید
    • یا از سرور محلی استفاده کنید: npx serve .

استقرار روی GitHub Pages

  1. فایل‌ها را در مخزن GitHub push کنید
  2. در Settings > Pages: Source = main branch / root
  3. آدرس سایت: https://username.github.io/repo-name/
