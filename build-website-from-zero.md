# آموزش ساخت صفر تا صد سایت شخصی و پنل مدیریت

این راهنما برای ساخت سایتی مشابه پروژه فعلی نوشته شده است: سایت شخصی فارسی، وبلاگ، گالری، پنل مدیریت، مدیریت صفحات، بنرها، PopUps، کاربران، فایل‌ها و تنظیمات.

## 1. تعریف نیازمندی

ابتدا محدوده را دقیق بنویسید:

- صفحات عمومی: صفحه اصلی، درباره من، تماس، وبلاگ، ورود و ثبت‌نام.
- پنل مدیریت: داشبورد، مدیریت صفحات، مدیریت محتوا، کاربران، گالری، وبلاگ، فایل‌ها، شبکه‌های اجتماعی و تنظیمات.
- ذخیره‌سازی: API backend برای محیط کامل و fallback لوکال برای محیط استاتیک.
- زبان و جهت: فارسی و `rtl`.
- طراحی: پنل مدیریتی کاربردی، کم‌تزئین، سریع و قابل اسکن.

## 2. ساخت Frontend

فایل‌های پایه بسازید:

- `index.html`
- `blog.html`
- `contact.html`
- `admin.html`
- `styles.css`
- `admin-styles.css`
- `script.js`
- `admin.js`
- `api.js`

در HTML عمومی، ساختار semantic، meta tagهای SEO، فونت فارسی و تصاویر اصلی را قرار دهید. در پنل مدیریت از sidebar، topbar، tab pane و sub tab استفاده کنید تا هر بخش جدا ولی سریع قابل دسترسی باشد.

## 3. ساخت Backend

برای backend از Express استفاده کنید:

- `server/server.js` برای راه‌اندازی API و static files.
- `server/db.js` برای اتصال دیتابیس.
- `server/routes/*.js` برای routeهای auth، users، posts، pages، banners، media، files، plugins، social و settings.

جدول‌های اصلی:

- `users`
- `pages`
- `posts`
- `categories`
- `tags`
- `post_tags`
- `banners`
- `media`
- `files`
- `settings`

## 4. پنل مدیریت صفحات

در بخش مدیریت صفحات، یک page builder داخلی مانند Elementor page builder با تمام ویجت‌ها و کارکردهایش بسازید: (حتما Elementor Page Builder Wordpress بررسی کامل شود)
حداقل بخش‌هایی که باید داشته باشد:

- لیست صفحات سایت
- ایجاد صفحه جدید
- canvas مرکزی
- navigator ساختار
- پنل تنظیمات محتوا، سبک و پیشرفته
- ویجت‌هایی مثل عنوان، متن، تصویر، ویدئو، دکمه، فرم، CTA، بنر، گالری، جدول قیمت و شبکه پست‌ها
- undo/redo
- preview دسکتاپ، تبلت و موبایل

## 5. مدیریت محتوا

تب جداگانه «مدیریت محتوا» شامل:

- بنرها: عنوان، تصویر، متن، CTA، محل نمایش، هدف‌گیری، زمان‌بندی، فعال/غیرفعال، حذف و آمار.
- PopUps: عنوان، محتوا، CTA، trigger، هدف‌گیری، زمان‌بندی، فعال/غیرفعال، حذف و آمار.
- آنالیتیکز: نمایش، تبدیل و نرخ تبدیل.

## 6. گالری

گالری باید:

- آپلود تصویر داشته باشد.
- اطلاعات نام، فرمت، حجم، ابعاد و تگ را نشان دهد.
- ویرایش inline داشته باشد، نه modal.
- انتخاب تصویر برای دیگر بخش‌های سایت را پشتیبانی کند.

## 7. وبلاگ

جدول وبلاگ باید ستون‌های زیر را داشته باشد:

- عنوان
- نویسنده
- تاریخ نگارش
- تاریخ انتشار
- دسته‌بندی
- وضعیت
- عملیات فوری: پاک کردن، در انتظار، انتشار، انتشار زمان‌دار
- ویرایش/حذف

## 8. تست

قبل از تحویل:

- ورود به پنل را تست کنید.
- تغییر tab و sub tab را تست کنید.
- ایجاد و ویرایش صفحه را تست کنید.
- ایجاد بنر و PopUp را تست کنید.
- ویرایش inline گالری را تست کنید.
- ذخیره و تغییر وضعیت پست را تست کنید.
- responsive موبایل را بررسی کنید.
- خطاهای JavaScript را با `node --check admin.js` بررسی کنید.
