# Backend Implementation Plan — Node.js + Express + MySQL

> **Goal:** Replace all localStorage operations with a real backend API connected to MySQL, with proper auth, file uploads, and security.

**Tech Stack:** Node.js, Express, MySQL2, bcryptjs, jsonwebtoken, multer, cors

## Architecture

```
Browser (HTML/CSS/JS)
    ↓ HTTP REST API
Express Server (server.js)
    ↓ MySQL2 driver
MySQL Database (database.sql)
```

## Files to Create

```
server/
├── server.js              ← Main Express server + routes
├── db.js                  ← MySQL connection pool
├── routes/
```

## Implementation Steps

### Step 1: Project Setup
- Create `package.json` with dependencies
- `npm install express mysql2 bcryptjs jsonwebtoken multer cors dotenv`
- Create `.env` with DB credentials
- Create `db.js` with MySQL connection pool

### Step 2: Database Schema Update
- Update `database.sql` to include proper password hashing fields
- Add `password_hash` column with bcrypt support
- Add JWT token table if needed

### Step 3: Auth System (server/routes/auth.js)
- POST /api/auth/login — validate credentials with bcrypt
- POST /api/auth/register — create user with hashed password
- POST /api/auth/reset-password — send reset token (simulated)
- JWT token generation and verification

### Step 4: Main Server (server/server.js)
- Express app setup with CORS, JSON parsing
- Mount all route modules
- Static file serving for the frontend
- Error handling middleware

### Step 5: Connect Frontend to API
- Replace all localStorage calls in admin.js with fetch() calls
- Replace auth calls in register.html/reset-password.html
- Add API base URL configuration

### Step 6: File Upload System
- Multer configuration for image/file uploads
- Upload to server filesystem
- Store file metadata in database
- Serve files via static middleware

### Step 7: Testing & Verification
- Test all API endpoints
- Test frontend-backend integration
- Test file uploads
- Test auth flow
- Test mobile responsiveness

### Step 8: Update SITE-PROMPTS.html
- Add backend architecture section
- Add API documentation
- Add deployment instructions
